1. Logo System

Primary Logo Lockup

The SocialSync logo consists of the wordmark "SocialSync" in Poppins Black, paired with the STAGE OTT logo and the tagline "Your Social Media Executive".

Horizontal Lockup (Preferred)

Usage Guidelines

  • Primary use: Application headers, website, marketing materials
  • Minimum width: 120px digital, 25mm print
  • Clear space: Minimum space around logo equal to the height of the "S" in SocialSync

Logo Variations

  • Full lockup: SocialSync + by STAGE + tagline
  • Short lockup: SocialSync + by STAGE (without tagline)
  • Wordmark only: SocialSync (for tight spaces)
05

Logo Usage Don'ts

Don't stretch or distort

Never alter the proportions of the logo. Always scale proportionally.

Don't change colors

Use only approved color variations. Never apply custom colors or gradients to the logo.

Don't add effects

No shadows, outlines, glows, or other effects. Keep the logo clean and crisp.

Don't rotate

The logo should always be displayed horizontally or vertically, never at angles.

Don't place on busy backgrounds

Ensure sufficient contrast. Use solid backgrounds or subtle gradients only.

Don't crop or rearrange

Keep all elements of the lockup together. Don't separate "by STAGE" or the tagline.

7. Motion & Animation

Animation Principles

Motion in SocialSync is purposeful, subtle, and enhances usability without distraction.

Duration

  • Micro interactions: 200ms (button hovers, icon changes)
  • Major transitions: 300ms (navigation, view changes)
  • Page transitions: 500ms (rare, significant changes)

Easing

cubic-bezier(0.4, 0, 0.2, 1) - Standard ease-in-out

Creates smooth, natural motion that accelerates and decelerates.

Common Animations

Hover Effects

  • Buttons: Scale to 105%, duration 200ms
  • Icons: Scale to 110%, duration 300ms
  • Cards: Scale to 105%, shadow intensification, duration 300ms
  • Links: Color transition, duration 200ms

Active States

  • Button press: Scale to 95%, duration 200ms
  • Navigation item: Background gradient + scale 105%
  • Pulse indicator: Fade opacity 50-100%, duration 1000ms, infinite

Loading States

  • Spinner: Rotate 360deg, duration 1000ms, infinite
  • Skeleton: Pulse opacity, duration 1500ms, infinite

Animation Guidelines

Use animations for feedback

Confirm actions, show state changes, guide attention

Don't overuse animations

Too many animations can be distracting and hurt performance

8. Voice & Messaging

Brand Voice

SocialSync communicates with a voice that is:

Professional

We're a reliable tool for serious creators and businesses. Our language is clear, accurate, and respects the user's time and intelligence.

Approachable

While professional, we're not stuffy or intimidating. We use conversational language that feels helpful and friendly.

Confident

We know what we're doing. Our messaging is direct and assured, without being arrogant.

Action-Oriented

We use active voice and clear verbs. Every message guides the user toward their next action.

Writing Guidelines

Do's

  • Use active voice: "Connect your account" not "Your account can be connected"
  • Be specific: "Post at 9:00 AM daily" not "Post regularly"
  • Use clear verbs: Connect, Schedule, Post, Delete, Retry
  • Keep it concise: Respect the user's time
  • Provide context: Explain why something matters

Don'ts

  • Don't use jargon or technical terms without explanation
  • Don't be overly casual or use slang
  • Don't blame the user for errors
  • Don't use passive voice
  • Don't be vague or ambiguous

10. Accessibility

WCAG 2.1 AA Compliance

SocialSync is designed to meet WCAG 2.1 Level AA standards, ensuring the application is accessible to users with disabilities.

Color Contrast

Combination Ratio Rating
White on Gray 800 (#191919) 14.3:1 ✓ AAA
Gray 300 on Gray 800 7.1:1 ✓ AAA
Success Green on Gray 800 4.8:1 ✓ AA
Error Red on Gray 800 4.2:1 ✓ AA
Warning Yellow on Gray 800 7.1:1 ✓ AAA
Info Blue on Gray 800 4.5:1 ✓ AA

Requirements

  • Normal text (16px+): Minimum 4.5:1 contrast ratio
  • Large text (18px+ or 14px+ bold): Minimum 3:1 contrast ratio
  • UI components: Minimum 3:1 contrast ratio

Keyboard Navigation

  • All interactive elements are keyboard accessible
  • Tab order follows logical flow
  • Focus states are clearly visible
  • Keyboard shortcuts available for common actions
  • Escape key closes modals and dropdowns

Screen Reader Support

  • All icons paired with text labels or ARIA labels
  • Status updates announced to screen readers
  • Form fields have associated labels
  • Error messages linked to form fields
  • Loading states announced
22

Accessibility Best Practices

General Guidelines

Provide text alternatives

All icons should have text labels or ARIA labels. Don't rely on icons alone to convey meaning.

Don't use color alone

Never use color as the only way to convey information. Combine with shape, text, or icons.

Maintain contrast

Ensure all text and UI elements meet minimum contrast ratios (4.5:1 for normal text).

Don't disable zoom

Users should be able to zoom to 200% without loss of functionality.

Use semantic HTML

Proper heading hierarchy, nav elements, buttons vs links, form structure.

Don't hijack scroll

Respect user preferences for scrolling behavior and animation.

Testing Checklist

  • ✓ Test with keyboard only (no mouse)
  • ✓ Test with screen reader (VoiceOver, NVDA)
  • ✓ Check color contrast with automated tools
  • ✓ Zoom to 200% and verify functionality
  • ✓ Test with Windows High Contrast Mode
  • ✓ Verify focus indicators are visible
  • ✓ Check form validation messages
  • ✓ Test with motion reduced (prefers-reduced-motion)

Resources

  • WCAG Guidelines: https://www.w3.org/WAI/WCAG21/quickref/
  • Contrast Checker: https://webaim.org/resources/contrastchecker/
  • Screen Reader: VoiceOver (macOS), NVDA (Windows)
  • Browser Extensions: axe DevTools, WAVE
